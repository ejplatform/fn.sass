    variables

    config 

    $config: () !default;

    Main configuration dictionary. You customize Mendeleev by setting values to this dictionary.

    config-base 

    $config-base: () !default;

    Configuration store that can be overridden by framework authors. Mendeleev searches for configuration first in $config, then in $config-base and finally in the internal default configuration $mendeleev-config. This dictionary is useful to override default values, while still keeping the user-facing $config map empty and easy to handle.

    mendeleev-config 

    $mendeleev-config: (
    // TYPOGRAPHY
    typography: (
        // Generic
        modules: (
            headings: true,
            paragraph: true,
            scale: true,
        ),
        // Typefaces
        typeface: null,
        mono: $font-stack-ibm-plex-mono,
        sans: $font-stack-exo,
        serif: $font-stack-ibm-plex-serif,
        cursive: $font-stack-system-sans,
        // Vertical rhythm and modular scale
        size: (
            sm: 16,
            md: 18,
            lg: 20,
        ),
        lead: 1.5,
        heading-lead: 1.250,
        scale: (
            sm: 9/8,
            md: (9/8 + 5/4) / 2,
            lg: 5/4,
        ),
        // Overridden properties
        body: (),
        paragraph: (),
        headings: (
            letter-spacing: -0.03em,
            font-weight: 900,
            color: $color-brand,
        ),
    ),
    // COLORS
    palette: (),
    contrasts: (),
    color: (),
    // RESPONSIVE
    breakpoints: (
        sm: (0, 30rem),
        md: (30rem, 48rem),
        lg: (48rem, 60rem),
        xl: (60rem, 9999rem),
    ),
    // ANIMATIONS
    animation: (
        duration: 300ms,
        timing-function: ease,
    ),
);

    Default configuration options. This maps contains all default values for configurations that can be overridden by the $config or $config-defaults mappings.

    utility-class 

    @mixin utility-class($name, $props) { ... }

    Create a utility class setting all properties defined in a map.

    The example bellow creates 3 different classes: the .uppercase and %uppercase-important sets all properties with !important, while the third %uppercase does not use !important.

    Parameters

    $name
      Required

    name of the utility class

    $props
      Required

    mapping with the properties defined by this utility

    Example

    Create a utility class.
    

    @include utility-class(uppercase, (text-transform: uppercase))

    get-type-scale 

    @function get-type-scale() { ... }

    Return a type scale mapping from the argument.

    If argument is a number provides a ratio-based scale. If argument is a list, convert to a map with the progression: ('1': first, '2': second, '3': third, ...etc) Argument can be one of the pre-defined type scales: very-narrow, narrow, medium, wide, very-wide.

    This function takes no parameters.
